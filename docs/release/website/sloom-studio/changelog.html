Release notes

Changelog

What changed in each Signal Loom build, newest first. Signal Loom is in open beta (0.9.x) — desktop (Windows / Linux) and Android share one version line. Dates are when each build was cut.

0.9.10 Current July 2026

The cross-device release: the phone ↔ browser Image baton pass now carries every stroke both ways, live, verified end-to-end on real hardware — and the commercial license activates on Android with the same offline key as desktop.

New

  • Cross-device drawing, for real. Draw on the phone, take over in a served desktop browser, draw there, take it back — the document, its layers, and every stroke travel both directions live. “Continue from your other device” cards pick a handed-off image up with its full layer stack.
  • License activation on Android. Settings → License accepts the same key as desktop, verified fully offline (with a pure-JS fallback for older WebViews), and unlocks the commercial exports on the phone.

Fixed & improved

  • Synced documents no longer render blank. The high-resolution compositor worker crashed in production builds and pinned the canvas to an empty cached composite; it is now a properly bundled module worker with a safe fallback, and synced pixels repaint the moment they arrive instead of waiting for your next brushstroke.
  • A connected browser's edits actually reach the phone. The phone's LAN relay dropped request parameters, so the edit-baton gate silently rejected every op a served browser pushed — and live-sync polls re-sent the whole event log each time. Both are one fix: the relay now forwards the full request.
  • Baton handoff snapshots no longer stack “continue on another device” onto their own titles on repeat handoffs, and handoff cards open from any storage backend.
0.9.9 July 2026

The Community + license release, plus the biggest Video/Paper feature batch yet: free for noncommercial use with a one-time commercial license, motion-comic bubbles and captions on the timeline, pro three-point editing with JKL shuttle, Premiere XML export, and an AI frame-fix tool in Paper.

New

  • Community edition + commercial license. Signal Loom is free for personal and noncommercial use. The one-time $17.99 commercial license (Settings → License; verified offline, one key for desktop and Android) unlocks the professional print-production exports — KDP, PDF/X-4, PDF/X-1a, IDML, and CMYK/spot — and removes the once-a-day Community notice. No watermarks on anyone's art, ever: exports carry invisible provenance metadata instead (PNG/JPEG tags, PDF Producer, video container comment).
  • Motion comics. Speech bubbles, thought bubbles, and caption boxes are first-class timeline clips — placed on the Program stage, keyframe-animatable, with full typesetting (font, colors, tail direction/length, line height, letter spacing, alignment) in the Inspector.
  • Pro editing quartet. JKL shuttle with 2/4/8× speeds, Space/Home/End/arrow transport, Source Monitor in/out marks with three-point insert (,) and overwrite (.), and ripple/roll trims — including Alt-drag pointer ripple on clip edges.
  • Timeline power tools. Markers (Shift+M), audio fades and crossfades, per-track locks, and collapsible drawer-style tracks that shrink to a slim strip with clip hints.
  • Premiere interchange. Export the active sequence as FCP7 XML from the Output tab — Premiere imports it via File > Import; file-backed media links directly.
  • Paper: AI Fix Frame. Right-click a panel → point the model at correct sibling panels, describe what's right and wrong, optionally marquee just a region, and repaint with any configured edit model. Plus: Send Page / All Pages to the Source Library from the page menu — a comic layout becomes motion-comic source material in one click.
  • Program Monitor sidebar tabs (Tools / Info / Output), a full-height opacity keyframe band, and Ctrl+Alt+↑/↓ opacity/volume nudges at the playhead.

Fixed & improved

  • Source Library reliability. Deleting a generated asset now sticks even while its node stays wired; cross-window sync no longer ships multi-MB payloads; the library stops re-reading every asset on each change; failed ingests are reported instead of vanishing.
  • Splash screen artwork now loads in the packaged app.
  • Panel layout fixes across workspaces: undocked panel snapping, the Video menu-bar dead band, and Flow zoom controls no longer buried under the Source Library.
0.9.8 July 2026

New Flow building blocks plus provider and editor fixes: a .slimg node that round-trips an image through the Image editor, a FLUX LoRA weights node, Color Palette / Color Swatch nodes, the BytePlus (Seedream) provider, and Google-via-Vertex fixes so nano-banana and Gemini text appear whenever you're signed in.

New

  • .slimg node. Save a connected image as an editable .slimg, open it in the Image editor, then feed the flattened result back into your Flow graph — with an explicit "Read disk" so the output refreshes exactly when you want.
  • LoRA Spec node. Build the JSON LoRA weights (path + scale, up to 3) that FLUX LoRA models accept, wired straight into a FLUX image node.
  • Color Palette + Color Swatch nodes. A master palette that guides image/video colour, plus labelled per-scene swatches you drag out of it.
  • BytePlus (Seedream) provider. First-party image generation via BytePlus ModelArk.

Fixed & improved

  • Google via Vertex. nano-banana image and Gemini text now appear whenever Vertex is authenticated — including projects set through a GOOGLE_CLOUD_PROJECT environment variable — and detecting / signing in auto-selects your project.
  • Collapsed nodes keep their wiring. Shrink a node to save space and its connected edges stay drawn instead of vanishing.
  • Node-category labels now show on normal-width monitors (not just ultra-wide), node titles wrap to two lines instead of clipping the unique part, and the Video workspace and Source Library are named consistently across the app.
0.9.7 June 2026

The cross-device release. Your phone and a desktop browser can now open the same live project, with a one-tap baton so edits never collide — plus a reworked in-window menu and a wave of editor polish.

New

  • Cross-device sync. Run the phone's LAN host and open the project in a desktop browser at the same time — Flow graphs, Paper pages, the Image canvas, and the source library all stay in step over your local network.
  • "Working here ⇄ Phone" baton. Only one device holds the edit lock at a time; hand it off from the top bar and the other device drops to a read-only mirror, so two screens never fight over the same change.
  • In-window menu. A new menu that lives inside the app window — switch between a compact menu button and a classic always-visible menu bar (replaces the unreliable Linux global-menu path).
  • Live presence. The top bar shows when another device is viewing the same project, instead of forcing a project open "read-only."

Fixed & improved

  • Served-session source images now load over a dedicated host-asset endpoint, and persisted state no longer trips a storage-quota cascade that broke thumbnails and exports.
  • Hardened the sync channels against a module-init test flake; build and test gates are green.
0.9.5 June 2026

Brush-and-tools wave: the Image editor gained the modifier gestures painters expect, every workspace got its own tool iconography, and your settings can now travel encrypted.

New

  • Brush straight-lines. Hold Shift to snap a point-to-point straight stroke with any brush-family tool.
  • Healing / clone source. Alt-click to set a source point for the spot-heal and clone tools.
  • Distinct tool icons per workspace. No more shared glyphs — Flow, Image, Paper, and Video each have their own unambiguous tool set.
  • Encrypted settings backup. Export and re-import your full configuration as an encrypted backup file.
  • Fuller Atlas image-model coverage. Mapped each Atlas image model's capabilities and request shape, with Flow nodes and generative-fill gating reconciled to match.

Fixed & improved

  • Audited tool-hotkey and gamepad coverage so every tool is reachable from the keyboard and a controller.
0.9.3 June 2026

Security and Android-ergonomics wave.

New

  • API keys encrypted at rest. Keys are sealed with the OS secure store on desktop (safeStorage) and WebCrypto on web/Android — still never sent to Sloom, now encrypted on disk too.
  • Collapsible toolbars to reclaim canvas space on small screens.

Android

  • Fixed silent failures in File Save / Open and the Save-As download fallback.
  • Corrected the Atlas image request shape (per-model source field; reference images attached correctly).
  • Volume keys can drive brush size; vertical swipes scroll past property sliders instead of snagging.
0.9.2 June 2026

Image-editor stability and performance.

Fixed & improved

  • Marquee / selection drawing is dramatically faster (the freehand lasso no longer drops to sub-1 fps).
  • Moving a selection now floats it to a new layer (non-destructive, Photoshop-style).
  • Zoom no longer flashes to a checkerboard until the next draw — the GPU blit is clamped and the canvas present is synchronized.
  • Removed the dead empty band under the top bar and added File ▸ New.
  • Tool hotkeys no longer fire while you're typing in a field or dialog.
  • Fixed a canvas repaint / present stall that froze the UI until the pointer moved in or out of the window.
  • Layer pixels persist correctly so saving a project no longer wipes the canvas.
0.9.1 June 2026

Fixed & improved

  • First Windows / Linux desktop downloads published to sloom.studio with an in-app About + update check.
  • Top bar stays a single row at 1080p and up; Linux Electron defaults to native Wayland + GPU for a 60 fps canvas.
  • Repaired the text tool (worker crash + click-to-place self-dismiss); retuned brush defaults with accurate previews.
0.9.0 June 2026 · first public beta

The first public beta: four connected workspaces — node-graph Flow, a layered Image editor with a stylus brush engine and generative fill, print-ready Paper layout (threaded text, baseline grid, CMYK swatches, speech bubbles, tables, text-on-path), and a multi-track Video timeline — all sharing one .sloom project and source library, on desktop and Android with a phone-as-LAN-host mode, gamepad support, and bring-your-own-key for every provider.

Signal Loom is independent software in active beta — expect frequent builds. The desktop app checks this version line for updates; on Android, store builds update themselves. Spot something off? support@sloom.studio.
