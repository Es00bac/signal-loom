02 · workspacessuite

Four workspaces. One project.

Each workspace is a genuine standalone tool — Image and Paper even save their own file formats (.slimg, .slppr) — and the .sloom project is the umbrella that unites all four. Switch tools without switching files: a clip you generate in Flow appears in Image, Video, and Paper automatically, from one shared project and source library.

flowworkspace
The center of gravity

Built around Flow.

The generative node graph is what Sloom Studio was designed around — wire providers into a graph once and every workspace downstream inherits the results.

A deliberate choice

AI is optional.

Paint in Image, lay out a book in Paper, cut in Video. Import your media or make it by hand — you can build an entire project with zero AI and zero keys.

No prescribed path

Use one. Or all four.

Each workspace is powerful alone; together is where they shine. There's no single prescribed way to use Sloom Studio.

03 · flow-demoreplay

Watch a node run.

A looping replay of the real Flow workflow, mocked in plain HTML — no video file. A sketch, a style reference, and a typed prompt get wired into Image Generation; the result is wired into an Envelope that already holds two assets, and lands in the shared Source Library the moment it arrives. (The reference thumbnails were generated with nano-banana over Vertex — the same call the app makes.)

04 · paper-demoreplay

Sculpt a comic page.

The Paper workspace, replayed: rule out panel frames, hold Ctrl to reveal vertex handles and drag an edge midpoint into a custom shape, interlock the neighbouring frame so the sculpt never traps white space (its art refits to the new bounds), set border weight and colour in millimetres, tint the page, and drop in a speech bubble with a draggable tail — Preflight keeps it print-ready.

05 · sfx-demoreplay

Make it LOUD.

Paper's Comic SFX designer: eight presets — BANG!, KAPOW!, SCREECH, WHIRRRR, BOOM!, CRASH!, ZAP!, SLAM! — then every parameter is yours: fill, stroke and shadow measured in millimetres, skew and rotation, letter tracking, and trailing copies that sell the motion.

06 · brush-demoreplay

A brush that listens.

The Image workspace's brush engine, replayed with a simulated stylus: pressure drives size and flow, tilt shades like the side of the lead, Colour Rate mixes foreground into background mid-stroke, and Shift rules dead-straight lines — on the GPU brush engine (WebGL2, dirty-rect updates).

07 · video-demoreplay

Cut it together.

The Video workspace, replayed with its real two-monitor layout: a clip from the Project Source Bin is previewed in the Source Monitor and sent to a lane with its V1 track chip, a second clip is dragged straight onto the Sequencer Timeline, a crossfade is built the real way — opacity keyframes, the first key dragged to 0% so the rubber-band line ramps V2 in over V1 — a score lands on the audio lane, and scale keyframes (◇) animate a push-in while the playhead scrubs the Program Monitor — playing two real Veo shots, the same character cut between two angles. (Shown in one of the app’s 20+ themes.)

08 · flow-runimage-gen × 4

Start with a consistency library.

The opening move of a project: a Flow graph that generates the low-level consistency assets — the recurring objects, props, and environments you reuse everywhere — so scenes stay visually consistent from shot to shot. It also tours most of the workspace.

flow · consistency-assets.sloom4 providers

Prompt and Color Palette / Swatch nodes drive Image Generation across four providers at once — Atlas (Seedream v4.5), BytePlus (bytedance/seedream), Google (gemini-3-pro-image) and Black Forest Labs (FLUX) — and the results collect into an Asset Package and the shared Source Library. Portals carry shared style and colour tokens across the graph; two Vision Verify nodes run a Gemini vision model to check each result against the reference (✓ TRUE / ✗ FALSE) into Value Monitors; and several nodes are collapsed to their minimal size while every connection stays wired.

13 · questionsmonitor

Asked, answered.

Do I need API keys — or AI at all — to use Sloom Studio?
No. Every workspace works with zero keys configured. Paint in Image, lay out pages in Paper, cut a timeline in Video, import any media you like. AI features simply wake up when you add a provider key — and stay out of the way when you don't.
What exactly does $17.99 buy?
The commercial license — a one-time key, sent by email, verified offline, yours forever. It unlocks the professional print-production workspace: KDP-ready output, conformant PDF/X-1a and PDF/X-4 export with real CMYK through an embedded ICC output intent and named spot colors as Separation plates, plus a structurally valid Adobe IDML package export (IDML import is not yet supported). It also covers print-ready PDF with bleed and trim, CMYK and spot swatches, and everything that ships during the 0.9.x beta at no extra cost. Because it’s buy-once-own-it, every pro export we land during the beta is yours at no extra cost. It also marks your exports’ metadata as licensed, removes the startup notice, and is the legal license once you start earning with your work. One key covers desktop and Android. Everything else — all four workspaces, webcomics, video renders, sharing — is already free. Model usage is separate: billed by your own AI provider, only when you choose to generate.
Where do my projects and keys live?
On your device. Projects are .sloom files you can copy, back up, and move like any other file. API keys are stored locally, and requests go from your machine straight to the provider — Sloom Studio has no servers of its own and collects no analytics.
Which AI providers are supported?
Eight cloud providers out of the box — Google Gemini (incl. Vertex AI), OpenAI and OpenAI-compatible endpoints, BytePlus (first-party Seedream), Atlas Cloud, Hugging Face, Black Forest Labs, Stability AI, and ElevenLabs — plus a lane for local and open models via your own self-hosted endpoint. Reference images and masks follow the model, not the menu: the same capability works wherever that model runs.
What about Android?
The Android build — same project format, same four workspaces, with S Pen, Samsung DeX, and the LAN host built in — is available now as a beta APK above, and is heading to the Samsung Galaxy Store and Google Play. The same license key covers desktop and Android.
Can I try it without paying?
Yes — just download it free, no account, no trial clock. Sloom Studio is free for personal and noncommercial use: learn it, make things, share them. It's also source-available under the PolyForm Noncommercial license, so you can read every line and build it yourself from GitHub. When you start earning from what you make with it, that's when the $17.99 license is due — it unlocks the pro print exports and funds a solo dev. Free to learn on, fair to pay for once it's paying you.
