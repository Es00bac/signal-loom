02・ワークスペーススイート

4つのワークスペース。ひとつのプロジェクト。

各ワークスペースは正真正銘、単体で完結したツールです——ImageとPaperは自前のファイル形式（.slimg、.slppr）まで持ち、.sloomプロジェクトはその4つをまとめる傘です。ファイルを切り替えずにツールを切り替えられます。Flowで生成したクリップは、共有のプロジェクトとソースライブラリを通じて、Image・Video・Paperに自動的に現れます。

flowワークスペース
すべての中心

Flowを中心に設計。

生成的なノードグラフこそ、Signal Loomが中心に据えて設計されたものです——グラフに一度プロバイダーを配線すれば、下流のすべてのワークスペースがその結果を受け継ぎます。

意図的な選択

AIは任意です。

Imageで描き、Paperで本のレイアウトを組み、Videoでカットする。素材を取り込むか、手で作るか——AIなし、キーなしでもプロジェクト全体を作り上げられます。

決められた道はない

ひとつだけ使う。全部使う。

各ワークスペースは単体でも強力で、組み合わせればさらに輝きます。Signal Loomの使い方に、唯一の正解はありません。

03・フローデモリプレイ

ノードの実行を見る。

実際のFlowのワークフローを、プレーンなHTMLだけでループ再現したもの——動画ファイルは使っていません。ラフスケッチ、スタイル参照、そして入力したプロンプトがImage Generationノードに配線され、結果はすでに2つのアセットを持つEnvelopeに配線され、届いた瞬間に共有のソースライブラリに現れます。（参照用のサムネイルはVertex経由のnano-bananaで生成——アプリが実際に行う呼び出しと同じものです。）

04・ペーパーデモリプレイ

コミックのページを組む。

Paperワークスペースを再現：コマ枠を引き、Ctrlを押して頂点ハンドルを表示し、辺の中点をドラッグして自由な形にし、隣のコマと連動させることで変形しても余白が生まれない（art側が新しい境界に自動で合わせます）、罫線の太さと色をミリメートル単位で設定し、ページに色を乗せ、ドラッグ可能な尻尾つきの吹き出しを配置——Preflightが印刷対応を保ちます。

05・SFXデモリプレイ

爆音にする。

PaperのコミックSFXデザイナー：8種のプリセット——BANG!、KAPOW!、SCREECH、WHIRRRR、BOOM!、CRASH!、ZAP!、SLAM!——そこから先はすべて自由自在。塗り・ストローク・影をミリメートル単位で調整し、傾斜と回転、文字トラッキング、そして動きを演出する残像コピーまで。

06・ブラシデモリプレイ

反応するブラシ。

Imageワークスペースのブラシエンジンを、模擬スタイラスで再現：筆圧が太さと流量を左右し、傾きが鉛筆の側面のようにぼかし、カラーレートがストローク中に前景色と背景色を混ぜ、Shiftで完全な直線を引ける——GPUブラシエンジン（WebGL2、差分領域更新）で動作。

07・ビデオデモリプレイ

つなぎ合わせる。

Videoワークスペースを、実際の2モニターレイアウトで再現：プロジェクトソースビンのクリップがソースモニターでプレビューされ、V1トラックチップを持つレーンに送られ、2つ目のクリップがシーケンサータイムラインに直接ドラッグされ、クロスフェードが実際の手順で作られます——不透明度のキーフレーム、最初のキーを0%にドラッグしてゴムバンドのラインがV2をV1の上にランプインさせる——スコアが音声レーンに乗り、スケールのキーフレーム（◇）がプッシュインをアニメーションさせながら、再生ヘッドがプログラムモニターをスクラブ——実際のVeoで撮った2カット、同じキャラクターを2つのアングルでカットしたものが再生されます。（アプリの20以上あるテーマのうちひとつで表示）

08・フロー実行画像生成 × 4

一貫性ライブラリから始める。

プロジェクトの最初の一手：低レベルの一貫性アセット——あちこちで再利用する繰り返し登場するオブジェクト、小道具、環境——を生成するFlowグラフで、カットが変わってもシーンの見た目の一貫性を保ちます。ワークスペースのほとんどをひと通り巡る内容にもなっています。

flow · consistency-assets.sloom4プロバイダー

プロンプトとカラーパレット / スウォッチノードが、4つのプロバイダー——Atlas（Seedream v4.5）、BytePlus（bytedance/seedream）、Google（gemini-3-pro-image）、Black Forest Labs（FLUX）——を同時に使ってImage Generationを駆動し、結果はアセットパッケージと共有のソースライブラリにまとめられます。ポータルはグラフ全体で共有のスタイルとカラートークンを運び、2つのVision VerifyノードがGeminiのビジョンモデルで各結果を参照と照合し（✓ TRUE / ✗ FALSE）、Value Monitorに渡します。そしていくつかのノードは折りたたまれた最小サイズのまま、すべての接続は保たれています。

13・質問モニター

質問、答えます。

Signal Loomを使うのに、APIキーやAIは必要ですか？
いいえ。すべてのワークスペースはキーを一切設定しなくても使えます。Imageで描き、Paperでページを組み、Videoでタイムラインをカットし、好きなメディアを取り込む。AI機能は、プロバイダーキーを追加したときだけ目を覚まし——追加しなければ一切邪魔をしません。
¥3,000でちょうど何が手に入りますか？
商用ライセンス——一度きりのキーがメールで届き、オフラインで検証され、永久にあなたのものです。プロ向けの印刷制作ワークスペースを解放します：KDP対応の出力、埋め込みICCアウトプットインテントを通じた実際のCMYKで適合PDF/X-1a・PDF/X-4を書き出し、名前付きスポットカラーをSeparation版として保持し、加えて構造的に有効なAdobe IDMLパッケージも書き出せます（IDMLインポートはまだ未対応です）。断ち落とし・トンボつきの印刷対応PDF、CMYKとスポットのスウォッチも含まれ、0.9.xベータの間に追加されるプロ向け書き出しはすべて追加費用なしであなたのものになります。書き出しのメタデータにライセンス済みと記録し、起動時の通知を消し、あなたの制作物が収益を生み始めたときの正式なライセンスにもなります。1つのキーでデスクトップとAndroidの両方をカバー。それ以外——4つのワークスペースすべて、ウェブコミック、動画の書き出し、共有——はすでに無料です。モデルの利用料は別で、生成を選んだときだけ、自分のAIプロバイダーに請求されます。
プロジェクトとキーはどこに保存されますか？
あなたの端末上です。プロジェクトは.sloomファイルで、他のファイルと同じようにコピー、バックアップ、移動ができます。APIキーはローカルに保存され、リクエストはあなたの端末からプロバイダーへ直接送られます——Signal Loomには自前のサーバーがなく、アナリティクスも収集しません。
対応しているAIプロバイダーは？
標準で8つのクラウドプロバイダーに対応——Google Gemini（Vertex AI含む）、OpenAIおよびOpenAI互換エンドポイント、BytePlus（純正Seedream）、Atlas Cloud、Hugging Face、Black Forest Labs、Stability AI、ElevenLabs——さらに、自前でホストするローカル・オープンモデル向けのレーンもあります。参照画像やマスクはメニューではなくモデルに従うので、そのモデルがどこで動いていても同じ機能が使えます。
Androidについては？
Android版——同じプロジェクト形式、同じ4つのワークスペース、Sペン・Samsung DeX・LANホストを標準搭載——はSamsung Galaxy StoreとGoogle Playに近日登場予定です。デスクトップ版は上記の通り、本日から利用できます。
支払わずに試すことはできますか？
はい——無料でダウンロードするだけで、アカウントも試用期限もありません。Signal Loomは個人・非商用利用なら無料です：学び、作り、共有してください。PolyForm非商用ライセンスの下でソースも公開されているので、すべてのコードを読んで、GitHubから自分でビルドすることもできます。作ったもので収益を得るようになったとき、そこで¥3,000のライセンスが必要になります——それがプロ向け印刷書き出しを解放し、一人の開発者の活動資金になります。学ぶのは無料、稼げるようになったら正当な対価を。
