# Sloom Studio 0.9.13 — audit repair and creative fidelity

Released July 18, 2026.

Sloom Studio 0.9.13 reconciles all 79 tracked findings from two independent correctness audits and follows that repair sprint with a substantial Image brush-library and stylus-fidelity upgrade.

## Highlights

- Multi-window saves, project switching, startup recovery, external opens, and dirty-document transitions now share explicit project authority and ordering.
- Flow runs remain attached to the workspace, inputs, dependencies, and usage record that started them. Cancellation, retry, cache, reusable-function, API Requester, typed-reference, proxy, and media-materialization paths were corrected.
- Portable and synchronized Paper workspaces now carry their tabs plus verified managed images, font bytes, licenses, and ICC profiles.
- Paper editor typography now shares export layout truth for exact managed and variable fonts, drop caps, leading, hard breaks, contextual wrapping, balanced lines and columns, ruby, emphasis marks, and tate-chu-yoko.
- Image dirty-close, immutable history, named snapshots, selection masks, Source Library durability, and brush preset isolation were corrected.
- Image now includes 173 professionally calibrated brushes in 22 collapsible sets, using deliberate pressure, velocity, tilt, rotation, texture, paint-load, color, wet-mixing, and Wacom dynamics.
- Stylus azimuth is no longer multiplied by altitude: physical 90-degree left/up/right direction changes produce full 90-degree orientation changes in the live brush preview and painted dabs.
- Video comic duration, fallback composition, browser export retry/cleanup, exact font use, and reusable comic placement were corrected.

## Compatibility

This remains a `0.9.x` beta release. Existing `.sloom`, `.slppr`, and `.slimg` projects remain supported; the repair work adds validation, recovery, and migration behavior rather than intentionally changing the authored formats.
