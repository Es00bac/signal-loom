# Source Library

Source Library は、Sloom Studio プロジェクトの共有メディアリポジトリです。読み込んだファイル、生成されたアセット、再利用可能なコンポーネント、外部リソースへの参照を保持します。Flow、Video、Image、Paper のすべてのワークスペースが Source Library から読み書きできます。

本章では、Source Library、Source Bin ノード、Generated Pool の概念、サイドバーモード、読み込みワークフロー、アイテムカード、ドラッグアンドドロップ、再生成、ストレージバックエンド、クロスウィンドウ同期、LAN 配信、Paper 統合について説明します。

## 基本概念

### Source Library

Source Library は、すべてのビンとアイテムをプロジェクトスコープで集めたものです。`.sloom` プロジェクトの一部として保存されますが、大きなアセットは通常プロジェクトファイルの外に保存され、パスまたは識別子で参照されます。

### Source Bin

Source Bin は、Source Library へのビューを表示する可視パネルです。プロジェクトには、種類、タグ、検索クエリでフィルタリングされた複数のビンを持つことができます。Flow の Source Bin ノードは、1 つの Source Bin アイテムを読み取ります。

### Generated Pool

Generated Pool は、Flow ノード、AI 操作、生成ツールによって作成されたアセット用の特別な領域です。生成されたアイテムは個別に追跡されるため、再生成したり、生成元ノードを探したり、古い生成物を削除したりできます。

## サイドバーモード

Source Bin サイドバーにはいくつかのモードがあります。

- **Thumbnails** — プレビューサムネイルのグリッド表示。
- **List** — 名前、種類、日付、サイズを含むコンパクトなリスト。
- **Metadata** — タグ、生成元、使用状況の列を含むリスト。
- **Binned** — ビンごとにグループ化。
- **Starred** — スター付きアイテムのみ。

モードはサイドバーのヘッダーメニューから切り替えます。

## ビン

ビンは Source Library 内でアイテムを整理します。デフォルトのビンには以下があります。

- **Imports** — ディスクから読み込まれたファイル。
- **Generated** — AI や Flow によって作成されたアセット。
- **Editor Assets** — あらかじめ作成されたタイトル、シェイプ、テンプレート。
- **Bookmarks** — ブックマークされたアイテム。

サイドバーで右クリックして **New Bin** を選択すると、カスタムビンを作成できます。アイテムをビン間でドラッグして整理します。

## 読み込みワークフロー

### ファイル読み込み

**ファイル > メディアを読み込み** を使用するか、ファイルを Source Bin にドラッグして読み込みます。対応する種類には以下が含まれます。

- 画像: PNG、JPEG、WebP、TIFF、AVIF、PSD、RAW（対応時）
- 動画: MP4、MOV、WebM、AVI、MKV（バックエンドによる）
- 音声: WAV、MP3、FLAC、OGG、M4A
- ドキュメント: PDF、TXT、Markdown
- 字幕: SRT、VTT
- パッケージ: ZIP、.slimg、.slppr
- スクリプト: .sloom-script

### Flow キャンバスへのドロップ

ファイルマネージャーからファイルを Flow キャンバスにドラッグします。Sloom Studio はファイルを Source Library に読み込み、それを参照する Source Bin ノードを作成します。

### ネイティブファイル読み込み

ネイティブビルドでは、Sloom Studio はファイルをスクラッチフォルダーにコピーせずに参照で読み込むことができます。これはディスクスペースを節約しますが、プロジェクトは元のファイルパスに依存します。ファイルが移動した場合は、**Repair** を使って再リンクします。

## アイテムカード

Source Bin の各アイテムはカードとして表示されます。カードには以下が表示されます。

- サムネイルまたはアイコン
- 名前
- 種類バッジ
- スター状態
- 生成アイテムの生成元インジケーター
- 使用回数

### 主要アクション

カードにカーソルを合わせると、主要アクションが表示されます。

- **Preview** — ビューアーで開きます。
- **Place** — アクティブなワークスペースに追加します。
- **Open** — デフォルトの外部アプリケーションで開きます。

右クリックすると、完全なコンテキストメニューが表示されます。

## コンテキストメニュー

Source Bin アイテムを右クリックすると、以下のアクションが表示されます。

- **Preview / Place / Open**
- **Locate Generator Node** — 生成アイテムの場合、作成した Flow ノードにジャンプします。
- **Regenerate** — 生成元ノードを同じ入力または更新された入力で再実行します。
- **Copy Reference** — ノードで使用する識別子またはパスをコピーします。
- **Metadata** — タグ、メモ、由来を表示・編集します。
- **Remove** — アイテムを Source Bin から削除します。**Delete File** を選ばない限り、基になるファイルは削除されません。

## ワークスペースへのドラッグアンドドロップ

Source Bin からアイテムをどのワークスペースにもドラッグできます。

- **Flow** — Source Bin ノードを作成するか、互換性のある入力に接続します。
- **Video** — Source Bin とタイムラインに追加します。
- **Image** — 画像を編集用に開くか、レイヤーとして配置します。
- **Paper** — 画像を画像フレームに配置するか、画像フレームを作成します。

ドラッグの動作はターゲットによって異なります。たとえば、画像を Video タイムラインにドラッグするとクリップが作成され、Paper ページにドラッグすると画像フレームが作成されます。

## 再生成

生成アイテムは Source Bin から再生成できます。

1. 生成アイテムを右クリックします。
2. **Regenerate** を選択します。
3. 元の入力を使用するか更新するかを確認します。

再生成は、プロンプト、モデル、シードを変更してアセットのバッチを更新したいときに便利です。

## 生成元ノードを探す

Flow で作成されたアセットの場合、Source Bin アイテムには生成元ノードへの参照が保存されます。アイテムを右クリックして **Locate Generator Node** を選択すると、Flow キャンバスがそのノードまでパンし、選択します。

## ストレージバックエンド

Source Library は複数のストレージバックエンドをサポートしています。

| バックエンド | 説明 |
|---------|-------------|
| Scratch | デフォルトのローカルスクラッチフォルダー。 |
| IndexedDB | Web ビルド用のブラウザベースストレージ。 |
| Native | Electron とモバイルのネイティブファイルシステムアクセス。 |
| Android | Android のメディアストアとアプリストレージ。 |
| Data / Blob | 一時アセット用のインメモリまたは Blob ストレージ。 |

アクティブなバックエンドはプラットフォームと設定によって異なります。一部のバックエンドにはサイズ制限やパフォーマンス上のトレードオフがあります。

### アセットの Hydrate

Hydration は、リモートまたは遅延バックエンドから参照アセットをアクティブなワークスペースにコピーします。たとえば、新しいマシンで開いたプロジェクトは、クラウドや LAN ストレージからアセットを Hydrate してからでないと編集できないことがあります。

## クロスウィンドウとネイティブ同期

Sloom Studio は、同じプロジェクトに対して複数のウィンドウをサポートしています。あるウィンドウで Source Library に変更を加えると、他のウィンドウに同期されます。これには以下が含まれます。

- 新しい読み込み
- 再生成されたアイテム
- 名前変更されたアイテム
- 削除されたアイテム

ネイティブ同期はファイルシステムウォッチャーを使用します。ブラウザ同期は、利用可能な場合、Broadcast Channel または Shared Worker を使用します。

### 修復

リンクされたアセットが移動、名前変更、削除された場合、Sloom Studio は欠落としてマークします。**Source Library > Repair** を使用してファイルを検索するか、手動で再リンクします。

## LAN 配信

Sloom Studio は、ローカルネットワーク経由で Source Library アセットを配信できます。これは以下の用途に便利です。

- デスクトップとタブレット間でアセットを共有する。
- 同じ Wi-Fi ネットワーク上で共同レビューを行う。
- 別のマシンにストレージをオフロードする。

LAN 配信は **設定 > プロバイダー > LAN Server** で有効にします。アプリは、接続デバイス用のローカル URL と QR コードを表示します。

### セキュリティ

LAN 配信はローカルネットワークに限定されます。信頼できないネットワークでは有効にせず、不要なときは無効にしてください。

## Paper 統合

Source Library は Paper ワークスペースと緊密に統合されています。

- Source Bin から画像をドラッグして画像フレームに配置します。
- Source Bin からフォントを読み込んで組版に使用します。
- .slppr ドキュメントを埋め込みドキュメントとしてリンクします。
- ページをフラット化画像として Source Bin に書き出し戻します。

Paper ドキュメントは、参照している Source Library アイテムを追跡します。Linked Assets パネルにはこれらの参照とその状態が表示されます。

## プロジェクトスナップショットと書き出し

### プロジェクトスナップショット

プロジェクトスナップショットは、Source Library とすべてのワークスペースの現在の状態をキャプチャします。スナップショットは以下の用途に使用します。

- マイルストーンを保存する。
- 安全に実験する。
- バージョンを比較する。

スナップショットは **ファイル > Project Snapshot** から作成します。

### Source Library の書き出し

すべてのアセットを含めてプロジェクトを共有するには:

1. **ファイル > アセットを書き出し** を選択します。
2. 書き出すアイテムまたはビンを選択します。
3. 宛先フォルダーを選択します。
4. Sloom Studio は、参照されているすべてのファイルをそのフォルダーにコピーします。

完全にポータブルなパッケージには、**ファイル > プロジェクトをパッケージ化** を使用します。

## 注意点

- 大きな Source Library は、プロジェクトの読み込み時間を遅くすることがあります。
- ブラウザビルドにはストレージクォータがあることがあります。
- ネイティブファイル参照は、ファイルが移動すると破損します。可搬性のためには Repair または Package Project を使用してください。
- LAN 配信のパフォーマンスはネットワーク速度に依存します。
- Generated Pool のアイテムは蓄積することがあります。古い生成物は定期的に削除してください。

## ベストプラクティス

- 早い段階でアイテムをビンに整理してください。
- 頻繁に使用するアイテムにはスターを付けてください。
- 説明的な名前とタグを使用してください。
- 生成元ノードを探すのではなく、Source Bin から再生成してください。
- 別のマシンに移動する前にプロジェクトをパッケージ化してください。
