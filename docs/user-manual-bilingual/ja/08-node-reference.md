# Flow ノードリファレンス

本章では、Flow ワークスペースで使用できるノードをカタログ形式で解説します。ノードはカテゴリ別にグループ化されています。各ノードについて、その機能、入力と出力、使用上の注意を説明します。すべてのノードがすべてのビルドで利用可能というわけではありません。一部は特定のプロバイダー、ライセンス、またはプラットフォームを必要とします。

## 生成

Generate カテゴリのノードは、プロンプトやその他の入力からメディアを作成します。

### 画像

設定された画像プロバイダーを使って、テキストプロンプトから画像を生成します。

- **入力:** prompt、negative prompt、width、height、seed、model、provider options。
- **出力:** image（Source Library 参照）。
- **使用例:** このノードの前にプロンプト組み立てノードを接続します。再現性のために seed 入力を使用します。

### 動画

テキストプロンプト、画像、または既存の動画から動画クリップを生成します。

- **入力:** prompt、source image/video、duration、resolution、seed、model。
- **出力:** video（Source Library 参照）。
- **使用例:** 一部のプロバイダーには画像キーフレームが必要です。推定コストは Usage Bar で確認してください。

### 音声

プロンプトから音声や音楽を生成します。

- **入力:** prompt、duration、tempo、model。
- **出力:** audio（Source Library 参照）。
- **使用例:** 背景音楽やサウンドデザインに便利です。

### コンポジション

複数のソースから合成画像または合成動画を生成します。

- **入力:** layers、blend mode、dimensions、background。
- **出力:** composite media。
- **使用例:** Video や Image ワークスペースに切り替えずに、簡単な重ね合わせに使用します。

## 入力・データ

フローにデータを取り込むノードです。

### テキストプロンプト

プレーンテキスト文字列を出力します。生成ノードの基本的な入力です。

- **入力:** text。
- **出力:** prompt string。
- **使用例:** String Template や Prompt Mixer と組み合わせて動的なプロンプトを作成します。

### 値

任意のタイプの単一の値を出力します。

- **入力:** value。
- **出力:** value。
- **使用例:** モデル名やファイルパスなどの定数に使用します。

### 数値

数値を出力します。

- **入力:** number。
- **出力:** number。
- **使用例:** seed、サイズ、カウンター、計算に使用します。

### カラーパレット

色値のリストを出力します。

- **入力:** colors、name。
- **出力:** palette。
- **使用例:** 画像生成や Paper の色入力に接続します。

### カラースウォッチ

単一の色値を出力します。

- **入力:** color。
- **出力:** color。
- **使用例:** コンポジションやスタイル参照での定数色として使用します。

### LoRA スペック

画像生成用の LoRA（Low-Rank Adaptation）仕様を定義します。

- **入力:** model name、weight、trigger text。
- **出力:** LoRA spec。
- **使用例:** LoRA に対応した画像または動画生成ノードに接続します。

### ラフスケッチ

スケッチをキャプチャまたは読み込みます。

- **入力:** image、strokes。
- **出力:** doodle image。
- **使用例:** image-to-image 生成のガイダンスとして使用します。

### 画像トリミング

画像を領域にトリミングします。

- **入力:** image、x、y、width、height。
- **出力:** cropped image。
- **使用例:** 生成用や一貫したフレーミング用に画像を準備します。

### .slimg

パッケージ化された Sloom 画像アセットを読み込みます。

- **入力:** file path または Source Bin 参照。
- **出力:** image layers。
- **使用例:** Image Editor から Flow に編集可能な画像を持ち込みます。

### ソースビン

Source Library からアイテムを読み取ります。

- **入力:** item reference。
- **出力:** item data（種類に依存）。
- **使用例:** 既存のアセットをフローに入力する最も一般的な方法です。

### アセットパッケージ

パッケージ化されたアセットバンドルを読み込みます。

- **入力:** package reference。
- **出力:** multiple assets。
- **使用例:** 共有アセットコレクションを展開します。

## リスト・エンベロープ

コレクションや構造化データを扱うノードです。

### 型付きリスト

すべてのアイテムが同じタイプのリストを作成します。

- **入力:** items。
- **出力:** list。
- **使用例:** ループノードやバッチ操作に供給します。

### エンベロープ

複数の名前付き値を 1 つの構造化オブジェクトにまとめます。

- **入力:** named fields。
- **出力:** envelope。
- **使用例:** prompt、seed、model などの関連値をグループ化します。

### エキスパンダー

リストまたはエンベロープを受け取り、フィールドを個別の出力に展開します。

- **入力:** list/envelope。
- **出力:** individual items or fields。
- **使用例:** リストをバッチ生成器の並列入力に変換します。

### リスト平坦化

ネストしたリストを 1 段階のリストに平坦化します。

- **入力:** nested list。
- **出力:** flat list。
- **使用例:** ネストしたループの出力を結合します。

### リスト長

リスト内のアイテム数を返します。

- **入力:** list。
- **出力:** number。
- **使用例:** 進捗インジケーターやループ条件を駆動します。

## フロー制御

実行順序と分岐を制御するノードです。

### RUN ME

実行を開始する手動トリガーです。

- **入力:** （なし）
- **出力:** trigger signal。
- **使用例:** フローの先頭に配置します。実行ボタンを押すと発火します。

### シンプルループ

リストを反復処理します。

- **入力:** list。
- **出力:** item、index、done。
- **使用例:** item 出力を生成器に接続します。done を使って後処理をトリガーします。

### While ゲート

条件が真である間ループします。

- **入力:** condition。
- **出力:** iteration、done。
- **使用例:** 再試行、ページネーション、収束アルゴリズムに使用します。

### 停止条件

条件が真になったときにループを停止します。

- **入力:** condition。
- **出力:** stopped signal。
- **使用例:** ループ本体の内部に配置して早期に抜けます。

### オン/オフスイッチ

下流の分岐を有効または無効にします。

- **入力:** enable、input signal。
- **出力:** output signal or none。
- **使用例:** 再配線せずに機能を切り替えます。

### フォークスイッチ

セレクターに基づいて入力を複数の出力の 1 つにルーティングします。

- **入力:** selector、input signal。
- **出力:** case outputs。
- **使用例:** 多方向分岐を実装します。

## ロジック・計算

計算と意思決定を行うノードです。

### ブール論理

AND、OR、NOT、XOR 演算を実行します。

- **入力:** boolean values、operator。
- **出力:** boolean result。
- **使用例:** 条件フラグを組み合わせます。

### If / Else

真偽条件に基づいてデータをルーティングします。

- **入力:** condition、true value、false value。
- **出力:** selected value。
- **使用例:** 条件付きデータフローを実装します。

### 比較

2 つの値を比較します。

- **入力:** a、b、operator。
- **出力:** boolean。
- **使用例:** While Gate や If / Else と組み合わせます。

### スイッチケース

値をケースと照合して出力を選択します。

- **入力:** value、cases。
- **出力:** matched value or default。
- **使用例:** 多くのケースでは、入れ子になった If / Else よりクリーンです。

### 計算

加算、減算、乗算、除算、剰余、累乗を実行します。

- **入力:** a、b、operator。
- **出力:** number。
- **使用例:** 基本的な計算に使用します。

### 数式

数式文字列を評価します。

- **入力:** expression、variables。
- **出力:** number。
- **使用例:** 変数を使った複雑な数式に使用します。

### フォールバック選択

最初の非空または有効な入力を出力します。

- **入力:** candidates。
- **出力:** first valid value。
- **使用例:** 一部の入力が失敗する可能性があるときにデフォルトを提供します。

### JavaScript

JavaScript スニペットを実行します。

- **入力:** inputs、script。
- **出力:** return value。
- **使用例:** カスタムロジック、データ変換、API 用の準備に使用します。

### Python

Python スニペットを実行します。

- **入力:** inputs、script。
- **出力:** return value。
- **使用例:** データサイエンス、テキスト処理、Python ライブラリが必要な場合に使用します。

### JSON クエリ

パスまたは JSONPath 式を使って JSON データをクエリします。

- **入力:** json、query。
- **出力:** result。
- **使用例:** API レスポンスからフィールドを抽出します。

### 正規表現パース

正規表現でテキストを解析します。

- **入力:** text、pattern。
- **出力:** matches、groups。
- **使用例:** 非構造化テキストから構造化データを抽出します。

### JSON ビルダー

入力から JSON オブジェクトを構築します。

- **入力:** named fields。
- **出力:** json string。
- **使用例:** API Requester 用のペイロードを準備します。

### HTML サンドボックス

HTML をレンダリングして出力をキャプチャします。

- **入力:** html、css、width、height。
- **出力:** image or html string。
- **使用例:** スタイル付きカード、チャート、プレビューを生成します。

### API リクエスター

HTTP リクエストを行います。

- **入力:** method、url、headers、body、timeout。
- **出力:** status、body、headers。
- **使用例:** 任意の REST API に接続します。

### SQL クエリ

データベースに対して SQL クエリを実行します。

- **入力:** connection、query、parameters。
- **出力:** rows。
- **使用例:** 対応するデータベースからデータを取得または更新します。

### CSV 相互変換

CSV データを読み書きします。

- **入力:** csv string or file。
- **出力:** list of records / csv string。
- **使用例:** スプレッドシートデータのバッチ処理に使用します。

### XML/YAML 相互変換

XML または YAML データを読み書きします。

- **入力:** xml/yaml string or file。
- **出力:** parsed data / serialized string。
- **使用例:** 設定ファイルやドキュメント交換に使用します。

## テキストツール

プロンプトやテキストを操作するノードです。

### 文字列テンプレート

テンプレート文字列に変数を埋め込みます。

- **入力:** template、variables。
- **出力:** formatted string。
- **使用例:** 動的なプロンプトを構築します。

### 正規表現置換

正規表現を使ってテキストを置換します。

- **入力:** text、pattern、replacement。
- **出力:** replaced text。
- **使用例:** テキストをクリーンアップまたは正規化します。

### プロンプト結合

複数のプロンプト断片を区切り文字で結合します。

- **入力:** fragments、separator。
- **出力:** combined prompt。
- **使用例:** 多くのソースからポジティブプロンプトを組み立てます。

### ネガティブプロンプト

ネガティブプロンプト文字列を出力します。

- **入力:** text。
- **出力:** negative prompt。
- **使用例:** 生成ノードに接続して除外したいコンテンツを指定します。

### プロンプトミキサー

重み付きで複数のプロンプトを混ぜます。

- **入力:** prompts、weights。
- **出力:** mixed prompt。
- **使用例:** スタイルやコンセプトをブレンドします。

## ストーリーツール

ナラティブや連続ワークフロー向けのノードです。

### ストーリー状態

ストーリービート間で状態を維持します。

- **入力:** initial state、updates。
- **出力:** current state。
- **使用例:** キャラクター、プロットポイント、シーンコンテキストを追跡します。

### シードシーケンサー

決定論的なシード列を生成します。

- **入力:** base seed、count。
- **出力:** seed list。
- **使用例:** 再現可能なバリエーションを作成します。

### 感情分析

テキストの感情を分析します。

- **入力:** text。
- **出力:** sentiment label、score。
- **使用例:** 生成コンテンツをフィルタリングまたはタグ付けします。

### 画像特徴抽出

画像から特徴やタグを抽出します。

- **入力:** image。
- **出力:** tags、embeddings。
- **使用例:** 視覚コンテンツからプロンプトを駆動します。

### セリフ分割

テキストブロックをセリフ行に分割します。

- **入力:** text、separator。
- **出力:** dialogue list。
- **使用例:** 漫画や動画の脚本を準備します。

## 再利用・レイアウト

フローを整理して再利用するためのノードです。

### 関数

再利用可能なサブグラフをカプセル化します。

- **入力:** 内部の Function Input ノードで定義されます。
- **出力:** 内部の Function Output ノードで定義されます。
- **使用例:** モジュール化され、共有可能なロジックを構築します。

### グループ

実行を変更せずに、視覚的にノードをグループ化します。

- **入力:** （なし）
- **出力:** （なし）
- **使用例:** 大きなフローを整理します。

### 関数入力マーカー

Function ノードの入力を定義します。

- **入力:** default value、type。
- **出力:** value。
- **使用例:** Function サブグラフの内部に配置します。

### 関数出力マーカー

Function ノードの出力を定義します。

- **入力:** value。
- **出力:** （なし、Function からエクスポートされます）
- **使用例:** Function サブグラフの内部に配置します。

### 仮想エイリアス

別のノードまたは変数へのエイリアスを作成します。

- **入力:** target。
- **出力:** target と同じ。
- **使用例:** 長い接続線を減らします。

### ポータルペア

接続を入力ポータルと出力ポータルに分割します。

- **入力:** input portal の signal。
- **出力:** output portal の signal。
- **使用例:** 大きなグラフを整頓します。

### 画像エディター

Flow から Image Editor ワークスペースを開いて画像を編集します。

- **入力:** image、edit description。
- **出力:** edited image。
- **使用例:** パイプラインの途中で手動または AI 編集を適用します。

## モニター・デバッグ

フロー実行を確認・検証するためのノードです。

### 値モニター

シグナルの現在値を表示します。

- **入力:** value。
- **出力:** （なし、表示専用）。
- **使用例:** 実行を停止せずにデータをデバッグします。

### ビジョン検証

ビジョンモデルを使って画像を基準と照合します。

- **入力:** image、criteria。
- **出力:** pass/fail、reasoning。
- **使用例:** 自動化パイプラインで出力をゲートします。

## 設定

設定と構成を操作するノードです。

### 設定

設定値を読み取るか上書きします。

- **入力:** key、default value。
- **出力:** value。
- **使用例:** フローからプロバイダー設定、デフォルト、プロジェクト設定にアクセスします。

## ノードの組み合わせ方

ほとんどのフローは、少数のパターンを使います。

1. **トリガー → 準備 → 生成 → 出力**
2. **リスト → ループ → 生成 → 収集**
3. **入力 → 検証 → 分岐 → 処理**
4. **Source Bin → 変換 → ワークスペースへ送信**

Function Library の組み込み例から始めて、ノードを独自の設定に置き換えていってください。
