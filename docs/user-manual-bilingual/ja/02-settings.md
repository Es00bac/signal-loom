# 設定

設定モーダルは、Sloom Studio がすべてのワークスペースでどのように動作するかを制御します。API キー、デフォルトモデル、インターフェース設定、入力割り当て、ブラシプリセット、ライセンス情報を保存します。設定はユーザープロファイルごとに保存され、セッション間で保持されます。一部の設定はプロジェクトごとに上書きできます。

設定は **ファイル > 設定** またはショートカット `Ctrl+,`（Windows/Linux）、`Cmd+,`（macOS）で開きます。モーダルは、Providers、Defaults、Interface、Keyboard、Gamepad、Brushes、License、Backup、OSS Licenses の各タブに分かれています。

## プロバイダーと API キー

Providers タブでは、AI およびメディアサービスの認証情報を入力します。API キーはローカルに保存され、通常はブラウザのローカルストレージまたは Electron のセキュアストアに保存されます。プラットフォームによって異なります。これらは Sloom Studio のサーバーには送信されず、選択したプロバイダーを呼び出すためにのみ使用されます。

サポートされているプロバイダーのキーフィールドは以下のとおりです。

| プロバイダー | キー名 | 用途 |
|----------|----------|----------|
| OpenAI | `openai` | GPT テキストモデル、DALL·E 画像生成 |
| Google Gemini | `gemini` | Gemini テキスト・画像モデル、Vertex レイアウト |
| Hugging Face | `huggingface` | Hugging Face 推論エンドポイント |
| ElevenLabs | `elevenlabs` | テキスト読み上げとナレーション |
| Black Forest Labs | `bfl` | Flux 画像生成 |
| Stability AI | `stability` | Stable 画像・動画生成 |
| Atlas | `atlas` | Atlas メディアサービス |
| BytePlus | `byteplus` | BytePlus メディアサービス |

各キーをフィールドに入力し、**保存** をクリックします。小さなインジケーターがキーが存在するかどうかを示します。キーを削除するには、フィールドをクリアして保存します。

### セキュリティに関する注意

- API キーを含む設定バックアップファイルは共有しないでください。
- 共有マシンでは、可能な場合はネイティブビルドを使用し、キーを OS キーチェーンに保存してください。
- ブラウザビルドではキーがローカルストレージに保存されることがあります。サインアウトするかブラウザプロファイルを共有する場合は、キーをクリアしてください。

## デフォルトモデル

Defaults タブでは、各操作タイプのデフォルトモデルを選択できます。

- **テキスト生成** — テキストノードとストーリーツール用のデフォルト LLM。
- **画像生成** — 画像生成ノード用のデフォルトモデル。
- **動画生成** — 動画生成ノード用のデフォルトモデル。
- **音声生成** — 音声・ナレーションノード用のデフォルトモデル。

デフォルトは、ノードがモデルを明示的に指定していない場合に使用されます。任意のノードの Inspector でデフォルトを上書きできます。

## Vertex 認証

Vertex Auth パネルでは、Google Cloud Vertex AI の認証を設定します。Vertex AI は Gemini レイアウト、一部の画像モデル、エンタープライズ展開に使用されます。

3 つの認証方法がサポートされています。

1. **gcloud CLI** — マシン上の既存の gcloud ログインを使用します。Sloom Studio は `gcloud auth print-access-token` を呼び出してトークンを取得します。
2. **Application Default Credentials（ADC）** — 標準の ADC パスを使用します。通常は `GOOGLE_APPLICATION_CREDENTIALS` 環境変数で設定されたサービスアカウント JSON です。
3. **Service Account JSON** — サービスアカウントキーファイルを貼り付けるかアップロードします。キーは安全に保存され、アクセストークンのリクエストに使用されます。

組織のセキュリティポリシーに合った方法を選択してください。Vertex 認証は、Vertex 固有のプロバイダーまたは機能を使用する場合にのみ必要です。

## プロバイダー固有のエンドポイントとオプション

一部のプロバイダーでは、カスタムエンドポイントとオプションが設定できます。Providers タブでプロバイダーカードを展開すると、以下が表示されます。

- **Base URL / Endpoint** — デフォルトの API エンドポイントを上書きします。プロキシ、オンプレミス展開、OpenAI 互換サービスに便利です。
- **Organization ID** — 組織ヘッダーが必要なプロバイダー向け。
- **Timeout** — 応答を待つ最大時間。
- **Retry Count** — 失敗したリクエストを再試行する回数。
- **Batch Size** — 1 回のバッチリクエストで送信するアイテム数。
- **Local Endpoint** — Ollama や vLLM などのローカル推論サーバー向け。
- **Android Accelerator** — 対応する Android ビルドでハードウェアアクセラレーションを有効にします。
- **Render Backend** — プロバイダープレビュー用に CPU、WebGL、またはネイティブレンダリングを選択します。

これらのオプションはカスケードされます。プロジェクトレベルの設定がユーザーデフォルトを上書きし、ノードレベルの設定がプロジェクトデフォルトを上書きします。

## インターフェーステーマと密度

Interface タブでは、Sloom Studio の外観を制御します。

### テーマ

Sloom Studio は複数のインターフェーステーマをサポートしています。デフォルトは長時間のメディア編集セッション向けのダークテーマです。利用可能なテーマには以下が含まれることがあります。

- Dark（デフォルト）
- Dark High Contrast
- Light
- Light High Contrast
- System（OS の設定に従う）

### 密度

密度は UI のコンパクトさに影響します。

- **Comfortable** — パディングが大きく、タッチターゲットが押しやすい。
- **Compact** — 一度に更多信息が表示され、大きなモニター向け。
- **Minimal** — 二次的なラベルを非表示にし、クロームを減らす。

密度は、Source Bin のサムネイル、タイムライントラックの高さ、パネル間隔にも影響します。

## アプリメニュースタイル

以下から選択できます。

- **Compact** — トップナビゲーションバーの 1 つのメニューボタン。
- **Menubar** — 従来のメニューバー。macOS では OS と統合されます。

この設定を変更すると、即座に反映されます。

## 言語とロケール

言語 / ロケール設定は以下を制御します。

- インターフェース言語。
- 日付、時刻、数値の書式。
- Paper のデフォルト組版と綴じ方向。
- 利用可能なフォントプリセットとスペルチェックの動作。

言語を変更しても、既存のドキュメントコンテンツには影響せず、UI ラベルと新規ドキュメントのデフォルトのみが変わります。

## キーボードショートカット

Keyboard タブには、すべてのコマンドとそのショートカットが一覧表示されます。以下が可能です。

- コマンドを検索する。
- 現在の割り当てを表示する。
- ショートカットをデフォルトにリセットする。
- ショートカット一覧を書き出す。

一部のショートカットはグローバルで、どのワークスペースからでも動作します。他はワークスペース固有で、そのワークスペースがアクティブなときのみ動作します。競合は赤色でハイライトされます。

## ゲームパッド割り当て

Gamepad タブでは、ゲームパッドのボタンと軸をコマンドに割り当てられます。各コマンドに対して以下を設定できます。

- ボタンプレス
- ボタンホールド
- 軸方向
- 修飾キーの組み合わせ

ゲームパッド割り当ては、キーボードショートカットとは別に保存されます。このタブでゲームパッド入力全体を有効または無効にできます。

## ブラシとトリミングプリセット

Brushes タブには、Image Editor と Paper ワークスペースで使用されるカスタムブラシとトリミングプリセットが保存されます。

ブラシでは以下を定義できます。

- サイズ、硬度、不透明度、フロー。
- ダイナミクス：筆圧、傾き、速度、ジッター。
- テクスチャとスキャター。
- ブレンドモード。

トリミングでは以下を定義できます。

- アスペクト比。
- 固定サイズ。
- 解像度と DPI。
- 裁ち落としとセーフマージン。

プリセットは書き出し・読み込みが可能で、プロジェクト間やユーザー間で共有できます。

## ライセンス

License タブには、現在のライセンス状態が表示されます。

- **Community** — 無料モード。機能リマインダーが表示されることがあります。
- **Commercial** — 有料モード。すべての機能がロック解除されます。

Commercial ライセンスを有効化するには:

1. Sloom Studio の Web サイトまたは正規販売代理店からライセンスキーを購入します。
2. **設定 > ライセンス** を開きます。
3. ライセンスキーを入力します。
4. **Activate** をクリックします。

ライセンスの検証はオフラインの Ed25519 署名を使用します。アプリはキーをローカルで検証するため、起動ごとに外部に問い合わせる必要はありませんが、公開鍵バンドルの初回ダウンロードにはインターネット接続が必要な場合があります。

### Commercial 書き出しゲート

一部の書き出し形式や高ボリューム操作は、Commercial ステータスによってゲートされています。Community モードでゲートされた書き出しを試みると、書き出しダイアログで制限事項が説明され、アップグレード方法が提示されます。

## 設定のバックアップと復元

Sloom Studio は設定を JSON ファイルに書き出すことができます。これは以下の用途に便利です。

- 新しいマシンへの移行。
- チーム内でのプロバイダー設定の共有。
- 破損したプロファイルからの復旧。

バックアップするには:

1. **設定** を開きます。
2. Backup タブに移動します。
3. **Export Settings** をクリックします。
4. JSON ファイルの安全な場所を選択します。

復元するには:

1. Backup タブに移動します。
2. **Import Settings** をクリックします。
3. JSON ファイルを選択します。
4. 復元するカテゴリ（providers、interface、shortcuts、gamepad、brushes、license）を確認します。

**警告:** バックアップファイルには API キーとライセンス情報が含まれることがあります。安全に保管し、共有しないでください。

## OSS ライセンス

OSS Licenses セクションには、Sloom Studio が使用するオープンソースソフトウェアが一覧表示されます。React、Vite、TypeScript、React Flow、FFmpeg、Lucide アイコンなどが含まれます。各エントリには以下が表示されます。

- ライブラリ名とバージョン。
- ライセンスタイプ。
- 完全なライセンステキストへのリンク。

このセクションは、コンプライアンスと透明性のために提供されています。Sloom Studio を再配布するか、そのコードを基に構築する場合は、ライセンス義務を慎重に確認してください。

## 設定のカスケード

設定は、優先度が低い順から高い順に以下のように解決されます。

1. アプリケーションのデフォルト。
2. 設定モーダルのユーザー設定。
3. `.sloom` ファイルに保存されたプロジェクト設定。
4. ノードまたはアセットレベルの上書き。

設定が下位レベルで上書きできる場合、UI には通常、デフォルトが使用されていることを示す小さなリンクまたはアイコンが表示されます。

## 設定のトラブルシューティング

| 問題 | 解決策 |
|---------|----------|
| API キーが保存されない | **保存** をクリックしたか確認してください。ブラウザビルドでは、ローカルストレージが無効化されていないか確認してください。 |
| プロバイダーが未承認と表示される | キー、エンドポイント、組織 ID を確認してください。プロバイダー固有のエラーは Usage Bar で確認できます。 |
| 言語が変更されない | 一部のラベルが前の言語のままになる場合は、言語変更後に Sloom Studio を再起動してください。 |
| ショートカットの競合 | **設定 > Keyboard** を開き、赤色でハイライトされた競合を解決してください。 |
| ライセンスの有効化に失敗する | キーが完全であること、時計が正しいことを確認してください。オフラインでキーが拒否される場合は、サポートにお問い合わせください。 |

設定関連のワークフローの詳細については、ワークスペース固有の章を参照してください。
