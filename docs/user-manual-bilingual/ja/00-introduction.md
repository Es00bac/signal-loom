# Sloom Studio 入門

Sloom Studio は、生成 AI メディアワークフローアプリケーションです。ノードベースの自動化キャンバス、ノンリニア動画エディター、レイヤー画像エディター、DTP ワークスペースを 1 つの環境に統合しています。目的は、画像・動画・音声・印刷用ドキュメントの生成・編集・公開までの再現可能なパイプラインを構築し、用途ごとに複数のツールを行き来する必要をなくすことです。

本マニュアルは、Sloom Studio をユーザーの視点から解説します。各ワークスペースの役割、インターフェースの構成、設定の場所、よくあるタスクの進め方などを説明します。アプリケーションは React、Vite、TypeScript で構築され、Electron デスクトップアプリ、ブラウザ、Android ビルドとして実行されます。ほとんどの機能はプラットフォーム間で共通ですが、ファイルシステムアクセス、Android アクセラレーター、LAN 配信などのネイティブ連携は、Sloom Studio の実行環境によって動作が異なります。

## Sloom Studio とは

Sloom Studio の中核はフロービルダーです。キャンバスにノードを配置し、接続して実行します。ノードはテキストプロンプトから画像を生成したり、プロンプトを組み合わせたり、リストを処理したり、外部 API を呼び出したり、小さな JavaScript や Python のスニペットを実行したり、データを他のワークスペースに送ったりできます。アセットが作成されると、Source Library に保存され、Video、Image Editor、Paper の各ワークスペースがそこから取り込んでさらに編集できます。

4 つのワークスペースは次のとおりです。

- **Flow** — ノードキャンバス。生成パイプラインの構築、反復作業の自動化、リストのバッチ処理、AI プロバイダーへの呼び出しを行います。
- **Video** — トラック、コンポジション、キーフレーム、効果、キャプション、書き出しプリセットを備えたノンリニアエディターです。
- **Image** — 選択範囲、マスク、調整レイヤー、ブラシエンジン、PSD 読み込みに対応したレイヤー画像エディターで、生成 AI ツールも内蔵しています。
- **Paper** — 縦書き日本語組版、スポットカラー、PDF/X 書き出しに対応した、複数ページドキュメント・漫画・印刷レイアウト用の DTP ワークスペースです。

上部のナビゲーションバーから、いつでもワークスペースを切り替えることができます。プロジェクトには 4 つのワークスペースすべてのアセットとシーケンスを含めることができ、あるワークスペースで作成したアセットは通常、他のワークスペースに送ることができます。

## 4 つのワークスペース

### Flow ワークスペース

Flow ワークスペースは自動化が行われる場所です。フローはノードの有向グラフです。各ノードは 1 つの操作を実行します。画像を生成したり、数式を評価したり、2 つの値を比較したり、リストをループしたりできます。実行ボタンを押すと、Sloom Studio は入力から出力へとグラフを評価し、要求された結果を生成するために必要なノードだけを実行します。

Flow は以下の用途に適しています。

- バリエーションを持ったプロンプトで数十〜数百枚の画像をバッチ生成する。
- ポジティブプロンプト、ネガティブプロンプト、LoRA 仕様、シードシーケンスを組み合わせた再利用可能なプロンプトテンプレートを構築する。
- HTTP API、SQL クエリ、CSV ファイル、XML/YAML 相互変換を使って外部サービスをつなぐ。
- 関数やグループを作成し、同じプロジェクト内で再利用したり書き出したりする。

Flow ワークスペースには、ノード追加用の下部ツールバー、よく使う編集アクション用のコンテキストメニュー、エラーやサイクル検出・レイアウト提案を報告する診断パネルがあります。

### Video ワークスペース

Video ワークスペースはトラックベースのノンリニアエディターです。Source Bin からソースをタイムラインのトラックにドラッグしてコンポジションを組み立てます。Source Monitor ではクリップをプレビューし、挿入前に In/Out ポイントを設定できます。Program Monitor では編集中のシーケンスの現在フレームを表示し、クリップ・テキスト・シェイプ・コマしっぽのインタラクティブな変形コントロールに対応しています。

Video は以下の用途に適しています。

- 短尺ソーシャル動画、レビューカット、アーカイブマスターの編集。
- キャプション、ナレーション、生成音声の追加。
- クロマキー、トリミング、ストローク、ブレンドモードなどのクリップ効果の適用。
- H.264、HEVC、ProRes、WebM VP9、GIF、画像シーケンスへの書き出し。

Video ワークスペースは JKL シャトル再生、リップル／ロール編集、キーフレームアニメーション、FCP7 XML 書き出しに対応しており、他のエディターへの移行に利用できます。

### Image Editor ワークスペース

Image Editor ワークスペースは、従来のビットマップエディターに似たレイヤー画像エディターですが、生成 AI ツールが組み込まれています。ペイント、レタッチ、合成、inpaint、outpaint、背景除去、アップスケールなどの生成操作が可能です。選択範囲、マスク、調整レイヤー、ブレンドモードは従来の画像エディターのように動作し、Generative Fill バーは AI プロバイダーに接続して内容に応じた編集を行います。

Image Editor は以下の用途に適しています。

- 写真のレタッチと合成。
- Flow、Video、Paper ワークスペース用のアセット作成。
- 選択範囲やマスクを活用した生成編集。
- PSD ファイルの取り扱いと、マスク・レイヤー・フラット化画像の書き出し。

### Paper ワークスペース

Paper ワークスペースは、正確なレイアウトが必要なドキュメント向けです。漫画、書籍、雑誌、Zine、印刷用 PDF などに対応しています。見開きページ、親ページ、余白と段組、ベースライングリッド、縦書き・ルビ・圏点・縦中横などの高度な日本語組版に対応しています。また、コマフレーム、吹き出し、思考の吹き出し、効果音デカール、ガターナイフなど、漫画特有のツールも備えています。

Paper は以下の用途に適しています。

- 漫画・マンガ制作。
- スポットカラーと PDF/X 準拠の複数ページ印刷ドキュメント。
- テキストスレッドと回り込みを使ったテキストと画像の混在レイアウト。
- PDF/X-4、PDF/X-1a、KDP パッケージ、IDML、CBZ、Web コミック画像セットへの書き出し。

## 初回起動時の言語選択ゲート

Sloom Studio を初めて起動すると、言語選択ゲートが表示されます。このゲートでは、メイン UI が表示される前にインターフェース言語とロケールを選択できます。この選択はメニュー、ダイアログ、日付と数値の書式、およびデフォルトの組版設定に影響します。後から **設定 > インターフェース > 言語 / ロケール** で言語を変更できますが、初回ゲートが最も速い方法です。

日本語のドキュメント、漫画、縦書きを扱う場合、言語ゲートで日本語を選択すると、綴じ方向や適切なフォントプリセットなど、Paper のデフォルトも日本語向けに設定されます。

## コミュニティ版と商用版のライセンス

Sloom Studio には 2 つのライセンスモードがあります。

- **Community** — 無料で利用できます。Community ビルドでは、ほとんどのコンテンツを作成・編集・書き出しできます。一部の書き出し形式や高ボリューム機能にはリマインダーまたは透かしが表示されることがあります。Community ライセンスは Ed25519 署名でオフライン検証され、初回セットアップ後はインターネット接続を必要としません。
- **Commercial** — 購入したライセンスキーが必要です。Commercial ビルドでは、ゲートされた書き出し形式のロックが解除され、リマインダーが非表示になり、生成出力の商用利用権が有効になります。ライセンス状態は **設定 > ライセンス** に表示されます。

ライセンスによってゲートされている機能がある場合、UI には明確なインジケーターが表示されます。書き出しダイアログでは、Commercial ライセンスを必要とする出力を生成する前に警告が表示されます。プロジェクトの出力を販売または配布する予定がある場合は、Commercial ライセンスを使用してください。

## 基本操作

### トップナビゲーションバー

トップナビゲーションバーはウィンドウの最上部に横断しています。左側には Sloom Studio のワードマークとワークスペースタブ **Flow**、**Video**、**Image**、**Paper** が表示されます。中央にはアクティブなワークスペースに応じて変わるツールスロットがあります。右側にはプロジェクトレベルのボタン、Usage Bar、ズームと全画面表示コントロール、アプリメニューがあります。

### ワークスペースタブ

ワークスペースタブをクリックすると、コンテキストを切り替えられます。各タブの状態は保持されるため、次のような使い方ができます。

1. Flow ワークスペースでフローを構築する。
2. Image に切り替えて生成したアセットを編集する。
3. Flow に戻って編集済みのアセットでグラフを再実行する。

### アプリメニュー

アプリメニューは 2 つのスタイルで表示できます。

- **Compact** — トップナビゲーションバーの 1 つのメニューボタンで、すべてのコマンドをドロップダウンで開きます。
- **Menubar** — 従来の File、Edit、View、Window、Help メニューを持つメニューバーです。macOS ではシステムメニューバーと統合されます。

Compact と Menubar の切り替えは **設定 > インターフェース > アプリメニュースタイル** で行えます。

### 下部ツールバー

Flow ワークスペースでは、下部ツールバーからノード挿入、レイアウトコマンド、実行コントロールに素早くアクセスできます。他のワークスペースでは、状況に応じて下部ツールバーが置き換わったり非表示になったりします。

### コマンドパレット

`Ctrl+Shift+P`（Windows/Linux）または `Cmd+Shift+P`（macOS）を押すと、コマンドパレットが開きます。パレットには利用可能なすべてのコマンドが一覧表示され、名前やキーワードで検索できます。設定の表示、診断の実行、ワークスペースの切り替え、ショートカットを覚えていないアクションの実行に最適です。

## プロジェクトファイル

Sloom Studio では、作業を整理するために複数のファイル形式を使用します。

| 拡張子 | 目的 |
|-----------|---------|
| `.sloom` | メインプロジェクトファイル。ワークスペースの状態、フローグラフ、動画タイムライン、Paper ページ、設定の上書き、ソースアセットへの参照を含みます。 |
| `.slimg` | パッケージ化された画像アセット。ラスターまたはベクター画像データ、レイヤー、マスク、メタデータを保持し、Sloom Studio で再編集できます。 |
| `.slppr` | パッケージ化された Paper ドキュメント。ページレイアウト、組版、リンク画像、印刷設定を保持します。 |
| `.sloom-script` | ポータブルな Flow スクリプト。プロジェクトに読み込んだり、ユーザー間で共有したりできます。 |

プロジェクトファイルは通常、大きなメディアを埋め込みません。代わりに、Source Library とスクラッチフォルダー内のアセットを参照します。プロジェクトを別のマシンに移動する場合は、**ファイル > プロジェクトをパッケージ化** または **アセットを書き出し** を使用して、参照されているすべてのファイルをポータブルなバンドルにまとめてください。

### 新規プロジェクトの作成

**ファイル > 新規プロジェクト** を選択するか、`Ctrl+N` / `Cmd+N` を押します。新規プロジェクトは、1 つの Flow ワークスペースと空の Source Bin から始まります。ワークスペースメニューから、開始用のシーケンス、画像、または Paper ドキュメントを追加できます。

### 開くと保存

既存の `.sloom` プロジェクトを開くには **ファイル > 開く**（`Ctrl+O` / `Cmd+O`）を使用します。保存するには **ファイル > 保存**（`Ctrl+S` / `Cmd+S`）を使用します。Sloom Studio は定期的にワークスペースの状態をリカバリファイルに自動保存し、アプリが予期せず終了した場合にセッションを復元します。

### 読み込みと書き出し

- **メディアを読み込み** — 現在のワークスペースを変更せずに、ファイルを Source Bin に追加します。
- **プロジェクトを書き出し** — `.sloom` ファイルとオプションのアセットバンドルを書き出します。
- **アセットを書き出し** — 選択した Source Bin アイテムを一般的な形式でフォルダーに書き出します。

### スクラッチフォルダー

スクラッチフォルダーは、生成および読み込まれたアセットがキャッシュされる場所です。**ファイル > スクラッチフォルダーを設定** で指定します。スクラッチフォルダーを高速なドライブにすると、大きな動画や画像のワークフローがスムーズになります。スクラッチフォルダーは、プラットフォームに応じてローカル、外部ドライブ、ネットワーク上に置くことができます。
