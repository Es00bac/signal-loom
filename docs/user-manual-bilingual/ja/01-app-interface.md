# アプリインターフェース

Sloom Studio のインターフェースは、各ワークスペースに適応する共有シェルを中心に構築されています。トップナビゲーションバー、アプリメニュー、ステータス領域は常に表示され、ウィンドウの中央部分は Flow キャンバス、Video タイムライン、Image Editor ビューポート、Paper レイアウトに切り替わります。本章では、共有のクローム、ワークスペース固有のツールバーとパネル、アプリケーション全体で利用できる入力方法について説明します。

## トップナビゲーションバー

トップナビゲーションバーはウィンドウの全幅にわたって表示されます。いくつかのゾーンに分かれています。

### ブランドとワークスペースタブ

左端には Sloom Studio のワードマークがあります。その隣にはワークスペースタブがあります。

- **Flow**
- **Video**
- **Image**
- **Paper**

アクティブなタブはハイライト表示されます。タブをクリックするとワークスペースを切り替えられます。また、`Ctrl+Tab` / `Cmd+Tab` と `Ctrl+Shift+Tab` / `Cmd+Shift+Tab` でワークスペースを循環させることもできます。

### ツールスロット

ナビゲーションバーの中央には、アクティブなワークスペースに応じて変わるツールスロットがあります。Flow ではズームコントロールや実行状態、Video ではトランスポートコントロール、Image と Paper ではアクティブなツールやクイック修飾子が表示されます。正確な内容はワークスペースと現在の選択によって異なります。

### プロジェクトと関数ボタン

ツールスロットの右側には、プロジェクトレベルのボタンがあります。以下が含まれることがあります。

- **Project Library** — アセット、関数、再利用可能なコンポーネントを管理するためのプロジェクトライブラリモーダルを開きます。
- **Function Library** — Flow 関数を再利用するための Function Library ドロワーを開きます。
- **Edit Baton** — 共同作業やコンテキスト切り替え時に、現在編集ロックを持っているユーザーを表示します。

### ズームと全画面表示

右端にはズームと全画面表示のコントロールがあります。ズームは、アクティブなワークスペースのビューポート（Flow キャンバス、Video ステージ、Image キャンバス、Paper ページ）に影響します。全画面表示はアプリを画面全体に広げ、OS のクロームを非表示にします。

### アプリメニュー統合

アプリメニューは、**設定 > インターフェース > アプリメニュースタイル** の設定に応じて、Compact スタイルのメニューボタン、またはフルメニューバーのいずれかで表示されます。

## ワークスペースタブ

ワークスペースタブは、4 つのワークスペース間を移動する主要な方法です。各タブは状態を保持するため、次のような使い方ができます。

1. Flow ワークスペースでフローを構築する。
2. Image に切り替えて生成したアセットを編集する。
3. Flow に戻って編集済みのアセットでグラフを再実行する。

ワークスペースを切り替えても、進行中のレンダリングは停止しません。動画書き出しや画像バッチ生成などの長時間実行される操作はバックグラウンドで継続され、Activity Trail で進捗が報告されます。

## アプリメニュー

アプリメニューにはメインアプリケーションコマンドが含まれます。構成は Compact スタイルか Menubar スタイルかによって異なります。

### Compact メニュー

Compact モードでは、トップナビゲーションバーの 1 つのメニューボタンを押すと、グループ化されたコマンドのドロップダウンが開きます。

- **ファイル** — 新規、開く、保存、メディアを読み込み、スクラッチフォルダーを設定、プロジェクトを書き出し、アセットを書き出し。
- **編集** — 元に戻す、やり直し、切り取り、コピー、貼り付け、削除、すべて選択、設定。
- **表示** — Source Bin の切り替え、Inspector の切り替え、ブックマークの切り替え、コマンドパレット、アクティビティ履歴、レイアウト初期設定、全画面表示。
- **ワークスペース** — Flow / Video / Image / Paper への切り替え。
- **ヘルプ** — ドキュメント、キーボードショートカット、OSS ライセンス、About。

### Menubar スタイル

Menubar モードでは、ウィンドウの上部に File、Edit、View、Workspace、Window、Help の個別メニューが表示されます。macOS では、これらのメニューがシステムメニューバーと統合されます。キーボードナビゲーションで `Alt` キーを使いたい場合は、Menubar スタイルを推奨します。

### クラシックメニューバー

`AppClassicMenuBar` は Menubar モードで使用される実装です。Compact メニューの項目を反映しつつ、可能な限りネイティブメニュー項目として表示します。各項目の右側にキーボードショートカットが表示されます。

## 下部ツールバー（Flow）

Flow ワークスペースでは、下部ツールバーがノードの追加とフローの制御の主要な手段です。以下が含まれます。

- **Add Node** — ノードカタログを開きます。カテゴリには、生成、入力・データ、リスト・エンベロープ、フロー制御、ロジック・計算、テキストツール、ストーリーツール、再利用・レイアウト、モニター・デバッグ、設定があります。
- **Run / Stop** — フロー評価の開始または停止を行います。
- **Clean Flow** — キャンバスを自動整理し、レイアウト上の問題をチェックします。
- **Zoom to Fit** — すべてのノードが見えるようにキャンバスを中央に配置し、スケールを調整します。
- **Add Source Bin Node** — Source Bin アイテムを参照するノードを作成します。

下部ツールバーはコンテキストに応じて変化します。現在の選択に適用されないボタンは無効化されます。

## コマンドパレット

コマンドパレットは、Sloom Studio のすべてのコマンドを検索できるリストです。次の方法で開きます。

- `Ctrl+Shift+P`（Windows/Linux）
- `Cmd+Shift+P`（macOS）

数文字入力するとフィルタリングされます。コマンドはワークスペースとカテゴリごとにグループ化されています。`Enter` を押すとハイライトされたコマンドを実行し、`Esc` で何も実行せずにパレットを閉じます。

便利なパレットコマンドの例:

- `Switch to Video`
- `Toggle Source Bin`
- `Open Settings`
- `Run Flow`
- `Clean Flow`
- `Export Project`
- `Command Palette` 自体

コマンドパレットは、コマンド名は覚えているがショートカットやメニュー位置を思い出せないときに特に便利です。

## アクティビティ履歴

Activity Trail パネルは、アクション、エラー、バックグラウンドタスクの時系列ログを記録します。ドッキング可能なパネルとして表示され、View メニューまたはコマンドパレットから切り替えられます。

各履歴エントリには以下が表示されます。

- タイムスタンプ
- ワークスペース
- メッセージ
- 重大度（info、warning、error）
- 該当する場合は関連アセットまたはノードへのリンク

エントリをクリックすると、そのソースに移動します。たとえば、失敗したノード実行のエラーをクリックすると、Flow キャンバスがそのノードまでパンし、Inspector が開きます。

## Usage Bar

Usage Bar はナビゲーションバーの右上エリアに配置されています。以下を表示します。

- 現在または次の AI 操作の**推定コスト**。
- 操作完了後の**実際のコスト**。
- クレジット残高を報告するサービスの**プロバイダー残高**。
- プロバイダー、モデル、操作タイプごとの**内訳**。

Usage Bar にカーソルを合わせると、完全な内訳がツールチップで表示されます。クリックすると詳細な使用状況レポートが開きます。Usage Bar は、有料 AI プロバイダー使用時の予期せぬ料金を回避するのに役立ちます。

## ダイアログ

Sloom Studio では、いくつかの共有ダイアログタイプが使用されます。

### アラートダイアログ

メッセージと **OK** ボタンを表示します。「書き出しが完了しました」や「ライセンスキーが受理されました」など、確認を求める情報に使用されます。

### 確認ダイアログ

**OK** と **キャンセル** ボタンを持つ質問を表示します。ノードの削除、Source Bin アイテムの削除、未保存プロジェクトの終了など、破壊的なアクションに使用されます。

### テキスト入力ダイアログ

テキスト値を求めます。新規プロジェクトの命名、ノードの名前変更、ライセンスキーの入力、簡易検索クエリの入力に使用されます。

### 高度なカラーピッカー

複数のカラーモデル、スウォッチ、スポットカラー選択をサポートするカラーピッカーダイアログです。Image、Paper、Flow の色値を受け入れるノードで使用されます。

### 設定モーダル

メイン設定ダイアログです。詳細は `02-settings.md` を参照してください。

### プロジェクトライブラリモーダル

関数、アセットパッケージ、テンプレートなど、再利用可能なプロジェクトコンポーネントを参照、読み込み、整理できます。

### 関数ライブラリドロワー

再利用可能な Flow 関数を一覧表示するスライドアウトパネルです。関数をキャンバスにドラッグすると、Function ノードが作成されます。

## ドッキング可能なパネル

Sloom Studio のほとんどのサイドパネルはドッキング可能です。以下が可能です。

- パネルのヘッダーをドラッグして、ウィンドウの左・右・下にドッキングする。
- パネルをタブで重ねる。
- パネルを独自のウィンドウに浮かせる（デスクトップのみ）。
- パネルの端をドラッグしてサイズを変更する。
- **表示 > レイアウト初期設定** でレイアウトをリセットする。

パネルレイアウトはワークスペースごとに保存され、プロジェクトを再度開いたときに復元されます。

### 共有ワークスペースドッキング可能パネル

`SharedWorkspaceDockablePanels` システムは、以下のような共通パネルを提供します。

- Source Bin
- Inspector
- Bookmarks
- Activity Trail
- Diagnostics
- Sequence Settings
- Export Preset

各ワークスペースは、そのコンテキストに適したパネルを有効にします。

## Source Bin サイドバー

Source Bin サイドバーには、プロジェクトの Source Library の内容が表示されます。2 つの主要モードがあります。

- **Compact** — サムネイルとクイックフィルターの狭い帯です。
- **Expanded** — 検索、並べ替え、種類フィルター、スター付きアイテム、メタデータを含む幅の広いパネルです。

Source Bin からアイテムを、Flow キャンバス、Video タイムライン、Paper ページ、Image Editor ビューポートにドラッグできます。アイテムを右クリックすると、プレビュー、配置、再生成、生成元ノードを表示、参照をコピー、削除などのアクションが表示されます。

## ブックマークサイドバー

Bookmarks サイドバーには、ブックマークしたノード、クリップ、レイヤー、ページが一覧表示されます。Flow では、ノードのブックマークを使って大きなグラフの重要な部分に素早くジャンプできます。Video では、タイムライン上の重要な位置をマークできます。Paper では、ページや見開きをマークできます。

ブックマークは、ワークスペース固有のブックマークコマンド（通常は `Ctrl+D` またはコンテキストメニュー）で追加します。削除は Bookmarks サイドバーから行います。

## ゲームパッド対応

Sloom Studio は、ナビゲーションや再生のためのゲームパッド入力に対応しています。ゲームパッドの割り当ては **設定 > ゲームパッド** でカスタマイズできます。一般的なデフォルトは以下のとおりです。

- 左スティック — ビューポートをパンする、または再生ヘッドを移動する。
- 右スティック — ズームまたはスクラブ。
- フェイスボタン — 再生/一時停止、In/Out のマーク、またはツール選択。
- ショルダーボタン — 前/次のフレームまたはノード。

ゲームパッド対応はオプションで、ソファでの閲覧、プレゼンテーション、アクセシビリティを想定しています。

## モバイルレイアウト

Android や幅の狭いブラウザウィンドウでは、Sloom Studio はモバイルレイアウトに切り替わります。

- トップナビゲーションバーがコンパクトなヘッダーに折りたたまれます。
- パネルがボトムシートになります。
- 下部ツールバーにタッチ操作に適したボタンが追加されます。
- コンテキストメニューがアクションシートになります。
- ピンチジェスチャーが一部のキーボードショートカットに代わります。

モバイルレイアウトでは、複数パネルのドッキングやフローティングウィンドウなど、一部の高度な機能が制限されたり利用できなかったりします。

## 初回起動とスタートアップ通知

### 初回起動時の言語選択ゲート

Sloom Studio を初めて起動すると、言語ゲートが表示されます。希望する言語とロケールを選択します。これは以下に影響します。

- メニューラベルとダイアログテキスト。
- 日付、時刻、数値の書式。
- Paper のデフォルト組版設定。
- 利用可能なフォントプリセット。

言語は後から **設定 > インターフェース > 言語 / ロケール** で変更できます。

### Community スタートアップ通知

Community ビルドでは、ライセンス条項と機能制限を思い出させるスタートアップ通知が表示されることがあります。この通知は通常、閉じることができ、Commercial ライセンスキーを入力するボタンが含まれていることがあります。Commercial ビルドでは、有効なライセンスが検出されるとこの通知をスキップします。

## Edit Baton

Edit Baton インジケーターは、Sloom Studio を共同作業または複数コンテキストのシナリオで使用している場合、現在編集ロックを保持しているユーザーを表示します。baton が別のユーザー名の横に表示されている場合、そのユーザーが baton を解放するか、あなたに移譲するまで、編集はキューに入れられたり制限されたりすることがあります。シングルユーザーモードでは、baton は常にあなたのものです。

## ヘルプコンテンツ

ヘルプコンテンツは Sloom Studio 全体に埋め込まれています。パネル、Inspector、ダイアログにヘルプアイコンを探してください。クリックすると、現在の機能の状況に応じた説明が開きます。ヘルプシステムは `interfaceThemes` および `helpContent` レジストリから取得するため、アクティブなテーマやロケールと整合性が保たれます。

## ナビゲーションショートカットのまとめ

| アクション | ショートカット |
|--------|----------|
| 次のワークスペースに切り替え | `Ctrl+Tab` / `Cmd+Tab` |
| 前のワークスペースに切り替え | `Ctrl+Shift+Tab` / `Cmd+Shift+Tab` |
| コマンドパレットを開く | `Ctrl+Shift+P` / `Cmd+Shift+P` |
| Source Bin の切り替え | ワークスペース固有、多くの場合 `Ctrl+B` |
| Inspector の切り替え | ワークスペース固有、多くの場合 `Ctrl+I` |
| Activity Trail の切り替え | ワークスペース固有 |
| 全画面表示 | `F11` または `Ctrl+Cmd+F` |
| 設定を開く | `Ctrl+,` / `Cmd+,` |

完全なショートカット一覧は `09-shortcuts.md` を参照してください。
