# Flow ワークスペース

Flow ワークスペースは、Sloom Studio の中核となるノードベースの自動化キャンバスです。このワークスペースでは、メディアを生成し、データを変換し、ロジックを評価し、リストをループし、外部サービスと通信するノードの有向グラフを構築します。Flow は、再現性を担う場所です。一度フローを構築すれば、新しい入力で再実行したり、リストに対してバッチ処理したり、関数として再利用したりできます。

本章では、Flow キャンバス、ノード操作、実行時の動作、診断、変数、バッチ実行、Source Bin 統合、インポート/エクスポート、ドラッグアンドドロップワークフローについて説明します。

## キャンバス操作

Flow キャンバスは無限の 2D 平面です。以下の方法で操作できます。

- **中クリックしてドラッグ** — キャンバスをパンします。
- **2 本指でピンチ** — トラックパッドやタッチスクリーンでズームイン・ズームアウトします。
- **スクロールホイール** — 垂直方向にスクロールします。`Shift` を押しながら水平方向にスクロールします。
- **キーボードズーム** — `Ctrl+=` でズームイン、`Ctrl+-` でズームアウト、`Ctrl+0` でリセットします。
- **Zoom to Fit** — 下部ツールバーまたはコマンドパレットから、すべてのノードを表示領域に収めます。
- **Minimap** — 隅の小さな概要図です。クリックするとその領域にジャンプします。

ノードをクリックして選択できます。`Shift` を押しながらクリックすると、選択に追加または削除できます。選択矩形をドラッグすると複数のノードを選択できます。`Ctrl+A` / `Cmd+A` ですべてのノードを選択できます。

## ノードの追加

ノードを追加するには、いくつかの方法があります。

1. **下部ツールバー** — **Add Node** をクリックしてノードカタログを開きます。
2. **中クリック検索** — キャンバスの空いている領域を中クリックして、ノードのクイック検索を開きます。
3. **コンテキストメニュー** — キャンバスを右クリックして **Add Node** を選択します。
4. **ドラッグアンドドロップ** — Source Bin アイテムをキャンバスにドラッグして、Source Bin ノードまたは互換性のある入力を作成します。
5. **コマンドパレット** — 「Add Node」または特定のノード名を検索します。

ノードカタログはカテゴリ別に整理されています。完全なカタログについては `08-node-reference.md` を参照してください。

## ノードの接続

ノードには左側に入力ハンドル、右側に出力ハンドルがあります。一部のノードには、制御フローまたは特別なルーティング用の追加ハンドルがあります。

2 つのノードを接続するには:

1. 出力ハンドルをクリックしてドラッグします。
2. 接続を互換性のある入力ハンドルにドロップします。

ドラッグした接続が近づくと、互換性のあるハンドルがハイライトされます。互換性のないハンドルは暗くなります。数値から文字列など、一致するか強制変換可能なタイプのみ接続できます。

接続を削除するには:

- 接続線をクリックして選択し、`Delete` を押します。
- 接続を右クリックして **Delete** を選択します。
- 接続をハンドルから離して放します。

## コンテキストメニュー

キャンバスまたは選択したノードを右クリックすると、状況に応じたコマンドのコンテキストメニューが開きます。

- **Add Node**
- **Copy / Cut / Paste**
- **Duplicate**
- **Delete**
- **Select All**
- **Add Source Bin Node**
- **Group Selected**
- **Create Function**
- **Bookmark Node**
- **Layout Defaults**
- **Clean Flow**

接続を右クリックすると、**Delete** または **Add Router** オプションが表示されることがあります。

## Clean Flow と自動整理

**Clean Flow** はキャンバスを再整理し、接続をまっすぐにし、診断を実行します。フローが散らかったときや、読みやすいグラフを共有したいときに便利です。

Clean Flow の実行時:

- ノードは依存関係に基づいて左から右の階層に配置されます。
- 長い接続線はノードの周りを迂回するように配線されます。
- 重なっているノードは分離されます。
- 切断されたグループは整列されます。

**注意:** Clean Flow は実行中、操作をブロックします。非常に大きなグラフでは数秒かかることがあります。希望の手動レイアウトを復元したい場合は、Clean Flow を実行する前にプロジェクトを保存してください。

Sloom Studio はまた、Vertex Gemini を使用して複雑なフローのレイアウトを提案することもできます。これは、Vertex 認証が設定されているときに **Clean Flow** のオプションメニューから利用できます。

## 診断

Diagnostics パネルは、現在のフローの問題を報告します。

- **Disconnected inputs** — 接続されていない必須入力。
- **Type mismatches** — 互換性のないタイプ間の接続。
- **Cycles** — 解決できないグラフ内のループ。
- **Unused outputs** — どの出力にも寄与しないノード。
- **Performance hints** — バッチ化や高コストノードの単純化に関する提案。

Diagnostics パネルは **View > Diagnostics** またはコマンドパレットから開きます。診断エントリをクリックすると、該当するノードまでキャンバスがパンします。

## 変数

Flow は、実行間またはノード間で値を保存する変数をサポートしています。変数はデフォルトで現在の Flow ワークスペースにスコープされますが、クロスワークスペースコマンドから読み取ることができます。

変数を作成または編集するには:

1. Variables パネルをワークスペースメニューまたはコマンドパレットから開きます。
2. **Add Variable** をクリックします。
3. 名前、タイプ、デフォルト値を入力します。

変数は、String Template、Math Expression、JavaScript、Python などの互換性のあるノードで名前で参照できます。名前を手動で入力する必要がないよう、ノードの Inspector には変数ピッカーがあります。

### 変数スコープ

- **Workspace scope** — 同じ Flow ワークスペースのすべてのノードで利用可能。
- **Function scope** — Function ノードの内部でのみ利用可能。
- **Loop scope** — Simple Loop や While Gate などのループノードが作成する一時変数。

名前には注意してください。内部スコープの変数はワークスペース変数をシャドウイングできます。

## バッチとリスト実行

多くの Flow ノードはリストを受け入れます。ノードがリストを受け取ったとき、アイテムごとに 1 回実行するか、実装に応じてリスト全体を一度に処理します。

### Typed List と Envelope ノード

- **Typed List** — アイテムタイプが固定されたリストを作成します。
- **Envelope** — 複数の値を構造化オブジェクトにまとめます。
- **Expander** — リストを受け取り、複数の出力に展開します。
- **List Flattener** — ネストしたリストを平坦化します。
- **List Length** — アイテム数を返します。

### ループ制御

- **RUN ME** — 実行を開始するトリガーノードです。
- **Simple Loop** — リストを反復処理し、各アイテムを出力します。
- **While Gate** — 条件が真である間、ループを続けます。
- **Stop When** — 条件が満たされたときにループを停止します。
- **On/Off Switch** — 分岐を有効または無効にします。
- **Fork Switch** — データを複数の出力のうち 1 つにルーティングします。

リストを実行すると、Activity Trail に各反復の進捗が表示されます。長いバッチ実行は、実行ボタンまたは Activity Trail からキャンセルできます。

### リストタイプの安全性

リストは型付けされています。文字列のリストを数値のリストを期待するノードに接続すると、タイプミスマッチの診断が発生します。変換ノードや Expander を使ってデータを整形してください。

## ノード実行モデル

**Run** を押すと、Sloom Studio はフローを評価します。

1. ノード接続から依存関係グラフを構築します。
2. サイクルを検出し、存在する場合は報告します。
3. 入力が変更されたために古くなったノードを特定します。
4. 依存順に古くなったノードを実行し、並列制限を遵守します。
5. 変更されていないノードが再実行されないよう、ノード出力をキャッシュします。
6. 結果を出力ノード、変数、または Source Library に書き込みます。

### シグナル評価

データはシグナルとして流れます。シグナルは値と、タイプ、エンベロープフィールド、プロバイダータグなどのメタデータを運びます。ノードは入力シグナルを読み取り、出力シグナルを計算し、次に渡します。Value Monitor などの一部のノードでは、現在のシグナル値を確認できます。

### 試行履歴

各ノードは、最近の実行、所要時間、エラーを示す試行履歴を保持します。ノードの Inspector を開き、**Attempts** セクションを展開して確認してください。

### 依存関係とキャッシュ

ノードは、入力のいずれかが変更された場合、または明示的に強制実行した場合にのみ再実行されます。ノードを右クリックして **Force Run** を選択すると、キャッシュ状態に関係なく再実行されます。これは、リモートファイルやデータベース行など、Sloom Studio が検出できない外部状態に依存するノードに便利です。

## Source Bin ノード

Source Bin ノードは Source Library からアイテムを読み取り、フローで利用できるようにします。ソースアイテムが変更されると、ノードは古くなり、再実行されます。

Source Bin ノードを作成するには:

1. Source Bin サイドバーからアイテムをキャンバスにドラッグします。
2. または、キャンバスを右クリックして **Add Source Bin Node** を選択し、アイテムを選びます。
3. または、ノードカタログから **Inputs & Data > Source Bin** を選択します。

Source Bin ノードは、ソースの種類に応じて画像データ、動画パス、音声パス、テキスト、またはドキュメント参照を出力できます。

## 複数の Flow ワークスペース

プロジェクトには複数の Flow ワークスペースを含めることができます。各 Flow ワークスペースには独自のキャンバス、変数、Source Bin ノードがあります。複数の Flow ワークスペースは以下の用途に使用します。

- 前処理パイプラインと生成パイプラインを分離する。
- 実験的なフローをメイングラフを乱すことなく保持する。
- 異なるフローを並列で実行する。

Flow ワークスペース間の切り替えは、ワークスペースタブエリアまたは Project Library モーダルから行います。

## インポートとエクスポート

### フローの読み込み

`.sloom-script` ファイルや別の `.sloom` プロジェクトのフローを現在のワークスペースに読み込むことができます。

1. **ファイル > 読み込み** を選択します。
2. ファイルを選択します。
3. 現在のワークスペースにマージするか、新しい Flow ワークスペースを作成するかを選択します。

### フローの書き出し

フロー全体を共有せずに一部を共有するには:

1. 書き出したいノードを選択します。
2. 右クリックして **Export Selected** を選択します。
3. `.sloom-script` として保存します。

プロジェクト全体を書き出すには、**ファイル > プロジェクトを書き出し** を使用します。

### ドラッグアンドドロップでの読み込み

ファイルマネージャーから `.sloom-script` または `.sloom` ファイルを Flow キャンバスにドラッグします。Sloom Studio は、ノードとして読み込むか、プロジェクト全体として読み込むかを確認します。

## ポータルの自動作成

接続が長くなったり大きな隙間を横切る場合、Sloom Studio は自動的にポータルペアを作成できます。ポータルペアは 1 つの論理接続を入力ポータルと出力ポータルに分割し、キャンバスを整頓します。

ポータルを手動で作成するには:

1. 接続を右クリックします。
2. **Create Portal** を選択します。
3. ポータルノードを都合の良い位置に移動します。

一方のポータルでの変更は、即座に他方に反映されます。

## ワークスペース指標

Flow ワークスペースでは、以下のような指標を表示できます。

- ノードと接続の数。
- 推定実行時間。
- キャッシュヒット率。
- プロバイダー推定コスト。

指標のオーバーレイは **View > Workspace Metrics** から開きます。

## 注意点とベストプラクティス

| 問題 | ガイダンス |
|-------|----------|
| Clean Flow が操作をブロックする | 大きなグラフで実行する前に保存してください。 |
| リストタイプの安全性 | リストタイプを一致させるか、明示的に変換してください。 |
| サイクル検出 | それ用に設計されたループノードを使用しない限り、フィードバックループを作らないでください。 |
| 変数スコープ | シャドウイングを避けるため、説明的な名前を使用してください。 |
| ブラウザ版とネイティブ版の違い | ファイルシステムノードはブラウザビルドで動作が異なることがあります。重いファイル I/O にはネイティブビルドを使用してください。 |
| キャッシュの予期せぬ動作 | ノードが外部状態に依存する場合は **Force Run** を使用してください。 |

## よくある Flow ワークフロー

### バッチ画像生成

1. ベースプロンプトを持つ **Text Prompt** ノードを追加します。
2. シード値またはプロンプトバリエーションの **Typed List** を追加します。
3. リストを **Simple Loop** に接続します。
4. ループ内で、プロンプトと現在のアイテムを **Image Generate** ノードに接続します。
5. フローを実行します。生成された画像は Source Bin に表示されます。

### プロンプトの組み立て

1. **Text Prompt**、**Negative Prompt**、**Prompt Mixer** ノードを追加します。
2. **LoRA Spec** ノードを接続してスタイルの重みを混ぜます。
3. **String Template** を使って変数を注入します。
4. 組み立てたプロンプトを画像または動画生成ノードに接続します。

### 外部 API の連携

1. **API Requester** ノードを追加します。
2. メソッド、URL、ヘッダー、ボディを設定します。
3. **JSON Query** または **Regex Parse** を使って結果を抽出します。
4. 結果を別のノードに渡すか、変数に保存します。

すべてのノードの完全なカタログについては `08-node-reference.md` を参照してください。
